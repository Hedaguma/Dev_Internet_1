Versiones de HTML

HTML 1.0

Publicado: 1991

HTML 1.0 fue la primera versión del lenguaje de marcado utilizado para crear páginas web. Creado por Tim Berners-Lee, el inventor de la web. Esta version contenia una conjunto basico de etiquetas que permitian a los desarrolladores a crear docuemtnos estructurados con enlaces y formatos de texto simples.

Ventajas:

  • Facil de aprender y usar
  • Comaptible con diferentes navegadores
  • Estandar de la web
  • Estructura y semantica
  • Compatible con otras tecnologias
  • Facilidad de mantenimiento

Desventajas:

  • Es muy limitado. En términos de etiquetas y características.
  • No hay soporte para multimedia o interactividad.
  • No existe un estándar claro para la presentación visual.
  • Vulnerable a errores de sintaxis

HTML 2.0

Disponible en: 1995

HTML 2.0 consolida y amplía las características de la versión original y las agrega nuevas etiquetas para mejorar la funcionalidad y estructura del contenido.

Ventajas:

  • Más estabilidad y estandarización.
  • Se introdujeron nuevas versiones, como tablas, formularios y texto consistente.
  • Amplio soporte en la industria

Desventajas:

  • Compatibilidad con multimedia y características complejas (gráficos).
  • Limitaciones en términos de interacción del usuario.
  • Vulnerabilidad a errores de sintaxis

HTML 3.2

Lanzamiento: 1997

Esta versión incluye tablas avanzadas, estilos y apariencia mejorada. La misma característica ha sido agregada. Este fue un gran paso hacia la web moderna.

Ventajas:

  • El diseño de la página mejoró gracias a tablas y otros elementos.
  • Introducción de soporte nativo de JavaScript.
  • Compatibilidad con diferentes navegadores

Desventajas:

  • Tecnologías no estándar, por ejemplo: Netscape personal
  • Limitada capacidad para diseñar interfaces interactivas y su vunerabilidad a errores de sintaxis

HTML 4.01

Entregado en: 1999

HTML 4.01 utiliza CSS para proporcionar estructura y presentación. Fue una mejora innovadora que introdujo la separación. La accesibilidad y la compatibilidad se identificaron como principios básicos.

Ventajas:

  • CSS mejoró enormemente la separación del contenido y la apariencia.
  • Se introdujeron reglas de accesibilidad como la propiedad ALT de las imágenes.

Desventajas:

  • Aún depende de la tecnología. No está completamente estandarizado como JavaScript.
  • Soporte multimedia limitado (requiere complementos como Flash).
  • Vunerabilidad a errores de sintaxis

XHTML

Lanzamiento: 2000

Se basó en la sintasix de XML, fue una importante evolucion de HTML y proporcionó una mayor modularidad y estructura al lenguaje.

Ventajas:

  • Es facil de mantener
  • Es facil editar el codigo directamente
  • Es mas estricto a nivel tecnico, lo que facilita la busqueda de errores y los cambios

Desventajas:

  • Su adopcion no fue tan grande como se esperaba
  • La mayoria de desarrollo de sitios web se centra en HTML5
  • Algunos navegadores antiguos no son totalmente compatibles con los estandares, lo que provacaba errores en las paginas.
  • Muchas herramientas de diseño web aun no producen codigo XHTML correcto

HTML5

Publicado: 2014

HTML5 es la versión más reciente y potente de HTML, que admite video, audio, animación, 2D y 3D. Soporte integrado para gráficos y más. Totalmente compatible con el diseño responsivo.

Ventajas:

  • Soporte multimedia integrado sin necesidad de complementos.
  • Mejor rendimiento y accesibilidad.
  • Nuevas funciones como almacenamiento web, geolocalización, lienzo y WebSockets introducidas.
  • Tiene un diseño móvil compatible y responsivo.
  • Facilita la optmizacion para motores de busqueda

Desventajas:

  • Limitada capacidad para diseñar interfaces interactivas
  • A veces se requieren conocimientos avanzados para aprovechar todo su potencial.

HTML 5.1 y 5.2

Lanzamiento: 2016 y 2017

Estas versiones son actualizaciones de HTML5 con correcciones, mejoras y más funciones de accesibilidad. Esta es una versión menor.

Ventajas:

  • Corrección de errores y mejoras de seguridad.
  • Actualizaciones continuas en términos de accesibilidad.

Desventajas:

  • Comparado con HTML5, más pequeño y más radical. El cambio puede verse como una falta de innovación.

HTML6

Lanzamiento:Indefinido

Es una version del lenguaje de marcado de hipertexto que permitira soportar hojas de estilo CSS4 y ser compatible con versiones anteriores.

Ventajas:

  • Permite migrar docuemtnos HTML4 a HTML6 y viceversa
  • Soporta hojas de estilo CSS4, lo que permite un diseño grafico mas visual
  • Es compatible con versiones anteriores de HTML.

Desventajas:

  • Puede tener problemas de accesibilidad y SEO si no se usa correctamente
  • Puede ser necesario complementar HTML con otros lenguajes para crear un sitio web completo.
  • Puede ser lento el proceso de diseño
  • Puede ser dificil corregir el codigo debido a que guarda mmuchas etiquetas.
